1. **Projekt verstehen:** [ANALYSE.md](./ANALYSE.md) lesen
2. **Umstrukturierung verstehen:** [UMSTRUKTURIERUNGSPLAN.md](./UMSTRUKTURIERUNGSPLAN.md) lesen
3. **Backend-Struktur:** [UMSTRUKTURIERUNG_BACKEND.md](./projekt/UMSTRUKTURIERUNG_BACKEND.md) lesen
4. **Frontend-Struktur:** [UMSTRUKTURIERUNG_FRONTEND.md](./projekt/UMSTRUKTURIERUNG_FRONTEND.md) lesen

### Für Tester

1. **Testkonzept:** [TESTKONZEPT.md](./testkonzept/TESTKONZEPT.md) lesen
2. **Testfälle:** Testfälle in [TESTKONZEPT.md](./testkonzept/TESTKONZEPT.md) durchgehen

### Für DevOps

1. **Docker Setup:** [DOCKER_SETUP.md](./setup/DOCKER_SETUP.md) lesen
2. **CI/CD:** GitHub Actions Pipeline prüfen
